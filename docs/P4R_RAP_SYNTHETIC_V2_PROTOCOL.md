# P4R RAP Fresh-Seed Boundary Confirmation

RAP-v1 remains a final NO-GO. It nevertheless produced positive held-out
confidence intervals in both domains and selected both maximum registered
profile energy and maximum decoder weight. This one permitted follow-up uses
new selection seeds 30--39 and new held-out seeds 40--59, none generated by
RAP-v1. The data-generating distributions, semantic-noise coupling, visible
message adversary, exact RDP accountant, sensitivity, matched GAP baseline,
selection criterion, and +0.02 Global/Cross gates are unchanged.

Only the profile-energy grid `[0.2, 0.35, 0.5]` and weight grid
`[1.0, 1.5, 2.0]` extend the two observed boundaries. This is the final
synthetic capacity check. Failure terminates RAP without real-data access;
success authorizes only a separately frozen real-data development protocol.
